# Plano de execução, portifolio Jorge Dias

## Objetivo
Criar um projeto Git chamado `portifolio`, com página pública gratuita no GitHub Pages, apresentando Jorge Dias em formato de currículo moderno para atrair recrutadores e clientes.

## Direção visual
Referência: Apple design-md, com inspiração em:
- muito espaço em branco
- tipografia limpa no estilo SF Pro/system font
- visual premium, minimalista e cinematográfico
- cards com glass, sombras suaves e foco em clareza
- efeitos visuais leves, sem pesar a página

## Entregáveis
1. Repositório Git local e remoto chamado `portifolio`.
2. Site estático publicado no GitHub Pages, sem custo de hospedagem.
3. Página com currículo, posicionamento profissional, habilidades, projetos, serviços e contato.
4. Contato com WhatsApp: `48 99208-7242`.
5. Pasta Obsidian dentro do projeto com arquivos Markdown vinculados.
6. Registro da referência de skills: `https://github.com/lucasmartinseti/skills`.
7. Código limpo, sem `node_modules`, sem build complexo e fácil de manter.

## Estrutura prevista
- `index.html`: página principal.
- `styles.css`: design visual.
- `script.js`: efeitos leves e interação.
- `README.md`: documentação do projeto.
- `docs/plano/`: planos de execução.
- `obsidian/`: vault/área lida pelo Obsidian.
- `obsidian/indices/`: mapas de conteúdo e links.
- `obsidian/portfolio/`: notas sobre currículo, posicionamento e site.
- `obsidian/skills/`: notas sobre skills e repositórios relacionados.

## Sequência de execução
1. Criar estrutura local do projeto.
2. Criar plano em Markdown.
3. Criar notas Obsidian sempre vinculadas por links internos.
4. Implementar site estático inspirado no design solicitado.
5. Inicializar Git.
6. Criar repositório GitHub `portifolio`.
7. Publicar GitHub Pages a partir da branch `main`.
8. Validar link publicado.
9. Entregar link final e resumo.

## Critérios de qualidade
- Página bonita, moderna e clara.
- Nada de hospedagem paga.
- Conteúdo com foco em recrutadores e clientes.
- Markdown sempre vinculado, sem nota solta.
- Projeto versionado e observável.
